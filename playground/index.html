Tokens de color

Muestra de tokens semánticos de color (superficies, contenido e intenciones: primary, success, warning, etc.).

Superficies base

--color-base-100
Superficie principal (background).
--color-base-200
Superficie secundaria (contenedores).
--color-base-300
Superficie elevada / paneles destacados.

Contenido base

Texto base
--color-base-content
Texto principal.
Texto muted
--color-base-content-muted
Texto de baja relevancia.
Texto subtle
--color-base-content-subtle
Texto de muy baja jerarquía.

Intenciones (brand & status)

--color-primary
Acciones primarias / highlights de marca.
--color-secondary
Acciones secundarias / énfasis alterno.
--color-accent
Detalles decorativos / acentos suaves.
--color-neutral
Backgrounds y contornos neutros.
--color-info
Mensajes informativos.
--color-success
Mensajes de éxito / confirmación.
--color-warning
Advertencias / riesgos moderados.
--color-error
Errores / estados críticos.

Tokens de bordes

Anchuras y estilos básicos de borde aplicados a superficies estándar.

--border-subtle
Borde estándar para cards, inputs y superficies neutras.
--border-base
Borde estructural más marcado (separadores fuertes).

Tokens de redondeo de esquinas

Radios semánticos para selectores, campos y superficies (todos con base 1rem en v1).

--radius-selector
Elementos pequeños de selección (badges, chips, toggles).
--radius-field
Inputs, botones, selects, tabs.
--radius-box
Cards, paneles y contenedores estructurales.

Tokens de elevación (sombras)

Niveles de profundidad 1–4 usando sombras semánticas --elevation-[n]-default.

--elevation-1-default
Elevación sutil, útil para estados hover o tarjetas muy ligeras.
--elevation-2-default
Elevación estándar para cards y paneles principales.
--elevation-3-default
Superficies flotantes (menús, paneles superpuestos).
--elevation-4-default
Modales y overlays de máxima prioridad visual.

Componentes — Contenedores y superficies

Cards, paneles y divisores que agrupan y jerarquizan contenido.

.card — levels (card-level-*)

Level 1

Neutral surface with minimal emphasis.

Level 2

Standard emphasis surface with elevation.

Level 3

Higher emphasis surface for highlighted content.

.card — sizes (card-size-*)

Size sm

Compact padding for dense UIs.

Size md

Default spacing for most surfaces.

Size lg

Spacious padding for prominent sections.

.card — variants (card-delimited, card-subtle, card-ghost)

Delimited

Strong border to clearly separate surfaces.

Subtle

Softer border for low-friction grouping.

Ghost

No border and no elevation for embedded sections.

.card-actions — orientation (card-actions-*)

Actions horizontal

Inline actions for compact layouts.

Actions vertical

Stacked actions for narrow containers.
